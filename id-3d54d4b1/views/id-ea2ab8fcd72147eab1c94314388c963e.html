SV-Healt Functional Architecture Principles ()
NARRATIVE: The SV-Healt Functional Architecture Principles view extends the Semantic view and provides a comprehensive framework for leveraging health-related data and promoting interoperability, reusability, and accessibility. It consists of several elements, their relations, and motivations: 1. Health Dataset Catalogue Catalogue (Grouping ABB): This element represents a catalog that contains various datasets related to health. It includes the following sub-elements: - Prescription catalogue (Data Object ABB): A dataset catalog that stores indexed inventory of prescription data, mitigating fragmentation through dispersed data sets. - Healthcare facilities catalogue (Data Object ABB): A dataset catalog that lists all accredited healthcare facilities. - Patient summary index (Data Object ABB): A dataset catalog that contains an indexed inventory of all patient summaries in a decentralized architecture. 2. Health Dataset Solutions (Grouping ABB): This element represents solutions related to health datasets. It includes various sub-elements: - Health synthetic metadata (Data Object ABB): A dataset containing artificially created metadata for research purposes. - Information to patients metadata (Data Object ABB): A dataset containing data related to information that can be directly provided to patients. - Diagnosis metadata (Data Object ABB): A dataset containing metadata for handling diagnosis processes and activities. - European patient summary metadata (Data Object ABB): A dataset containing metadata for managing patient summaries. - Health ERP (Enterprise Resource Planning) metadata (Data Object ABB): A dataset containing metadata for managing health processes in hospitals. - Caregivers metadata (Data Object ABB): A dataset containing metadata for collecting data about caregivers. - Integrated care pathways metadata (Data Object ABB): A dataset containing metadata and information related to different care pathways. - Patient summary metadata (Data Object ABB): A dataset containing metadata for managing patient summary processes and activities. - Relatives metadata (Data Object ABB): A dataset containing metadata for collecting data about relatives of patients. 3. Health Base Registry Catalogue (Grouping ABB): This element represents a catalog that contains various base registries related to health. It includes the following sub-elements: - Drugs interactions registry (Data Object ABB): A base registry containing all available drugs interactions. - Authorized treatments registry (Data Object ABB): A base registry containing a list of all possible legally authorized treatments. - Authorized medical images registry (Data Object ABB): A base registry containing a list of all types of medical images that can be legally prescribed or taken. - Authorized medical equipments registry (Data Object ABB): A base registry containing a list of all legally authorized medical equipments. - Authorized medical devices registry (Data Object ABB): A base registry containing a list of all legally authorized medical devices. - Imaging bank for test and experimentation facilities (Data Object ABB): A dataset containing medical images for testing and experimental purposes. - Credentials registry (Data Object ABB): A base registry containing credential data to access public administration information systems. - Health professional qualifications registry (Data Object ABB): A base registry containing a list of qualifications associated with healthcare professionals. - Diagnosis registry (Data Object ABB): A base registry containing all diagnoses performed by authorized health professionals. - ADT (Admission Discharge and Transfer) registry (Data Object ABB): A base registry containing data about patient admissions, discharges, and transfers. - Authorized vaccines registry (Data Object ABB): A base registry including all types of vaccines and vaccination treatments that can be legally dispensed. - Eligible patients registry (Data Object ABB): A data repository for researchers to submit their data, with specific requirements concerning subject or research domain. - Hospital tickets registry (Data Object ABB): A base registry containing data about daily processed patient hospitalization and consultation tickets. - Prescriptions registry (Data Object ABB): A base registry containing data about prescriptions issued by health professionals or healthcare facilities. - Laboratory analyser data registry (Data Object ABB): A base registry containing data about chemicals and characteristics in biological samples analyzed by laboratory analyzers. - Laboratory pre/post-analytical device data registry (Data Object ABB): A base registry containing data exchanged between an Automation Manager and automated Laboratory Devices for processing work orders and retrieving results. - Treatments registry (Data Object ABB): A base registry containing data about treatments prescribed to inpatients or outpatients. - Vaccinations registry (Data Object ABB): A base registry containing data about vaccinations performed by the national health system. - Patients registry (Data Object ABB): A base registry containing patient administrative data, such as demographic data and hospitalization data. - Healthcare payments registry (Data Object ABB): A base registry containing data about payments made by patients to access healthcare services. - Healthcare facilities registry (Data Object ABB): A base registry containing data about healthcare facilities. - Dispensation reports registry (Data Object ABB): A base registry containing data about medicines to be dispensed or that have been dispensed. - Laboratory reports registry (Data Object ABB): A base registry containing data about laboratory reports performed by patients. - Medical images registry (Data Object ABB): A base registry containing data about medical images captured in hospitals, clinics, or other legally entitled sites. - Patient summary registry (Data Object ABB): A base registry containing national patient summaries of patients. - Healthcare bills registry (Data Object ABB): A base registry containing data about bills related to healthcare services. 4. Health Ontology Solutions (Grouping ABB): This element represents solutions related to health ontologies. It includes various sub-elements: - International Classification of Diseases for Oncology (ICD-O) (Data Object SBB): An ontology for tumor diseases widely used by cancer registries. - Logical Observation Identifiers Names and Codes (LOINC) (Data Object SBB): A database and universal standard for identifying medical laboratory observations. - International Classification of Diseases (ICD) (Data Object SBB): A diagnostic tool used to classify and monitor causes of injury and death. - Anatomical Therapeutic Chemical Classification Ontology (ATCCO) (Data Object SBB): An ontology for classifying active ingredients of drugs. - International surgery procedure ontology (Data Object SBB): An ontology for the joint development of surgical ontologies. 5. Health Controlled Vocabulary Solutions (Grouping ABB): This element represents solutions related to health controlled vocabularies. It includes various sub-elements: - International Classification of Diseases for Oncology (Data Object SBB): An ontology for joint development of disease-related ontologies. - Systematised Nomenclature of Medicine (SNOMED) (Data Object SBB): A structured dictionary used in electronic medical record systems. - International consumer health terminology (Data Object SBB): An international dictionary of health terminology. - International Clinical Terminology for Primary Care (Data Object SBB): A set of definitions of terms describing the process of primary care. - Logical Observation Identifiers Names and Codes (LOINC) (Data Object SBB): A database and universal standard for identifying medical laboratory observations. 6. Data Interoperability (Principle SBB): This principle emphasizes the importance of ensuring interoperability between different health-related data sources. 7. Data Reusability (Principle SBB): This principle promotes the reusability of health-related data for various purposes. 8. Loosely coupled integration (Principle SBB): This principle advocates for loosely coupled integration between different health information systems. 9. Data Accessibility (Principle SBB): This principle emphasizes the need for easy access to health-related data. 10. Data Findability (Principle SBB): This principle focuses on making health-related data easily discoverable and searchable. 11. Data Portability (Principle SBB): This principle promotes the ability to transfer health-related data between different systems. 12. EU Localisation Framework compliance (Principle SBB): This principle emphasizes compliance with the EU Localisation Framework for health-related data. 13. Once Only (Principle SBB): This principle promotes the concept of capturing health-related data only once and reusing it across different systems. 14. Preservation of information (Principle SBB): This principle emphasizes the need to preserve health-related information for future use. 15. Transparency of internal information systems (Principle SBB): This principle promotes transparency in the functioning of internal health information systems. 16. Transparency of the administrative environment (Principle SBB): This principle promotes transparency in the administrative environment of healthcare systems. 17. Reuse, before buy, before build (Principle SBB): This principle encourages the reuse of existing solutions before considering buying or building new ones. 18. Health Dataset Catalogue Solutions (Grouping ABB): This element represents solutions related to health dataset catalogues. It includes the following sub-element: - European health professional qualification catalogue ISCO-08 (Data Object SBB): A European data catalogue containing a list of all health professional habilitations. 19. Health Base Registry Solutions (Grouping ABB): This element represents solutions related to health base registries. It includes the following sub-element: - European Reference Laboratory Network for TB (ERLTB-Net) (Data Object SBB): A network supporting countries in providing reliable and timely diagnostic services for tuberculosis. 20. Health Controlled Vocabulary Catalogue (Grouping ABB): This element represents a catalog that contains various controlled vocabularies related to health. It includes the following sub-element: - National specific taxonomy (Data Object ABB): A taxonomy consisting of term names and labels specific to public service information.
