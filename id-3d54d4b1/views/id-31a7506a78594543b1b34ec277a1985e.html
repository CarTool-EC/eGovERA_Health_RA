SV-Health Controlled Vocabularies [Motivation] ()
NARRATIVE: The SV-Health Controlled Vocabularies [Motivation] view extends the Semantic view and provides a comprehensive framework for leveraging various controlled vocabularies in the health domain. The view consists of two main groupings: "Health Controlled Vocabulary Solutions" and "Health Business Information Base". 1. Health Controlled Vocabulary Solutions: - National specific taxonomy: This is a Data Object ABB (Abbreviation) that represents a taxonomy specific to public service information in a particular country. It consists of term names and labels that are specific to the country's healthcare system. - International Classification of Diseases for Oncology: This is a Data Object SBB (Sub-Block) that represents an ontology used for the joint development of ontologies in the field of diseases. It provides a framework for classifying and organizing information related to diseases. - Systematised Nomenclature of Medicine (SNOMED): This is a Data Object SBB that represents the SNOMED system, which is a large structured dictionary used in electronic medical record systems. It provides standardized labels for storing information about patients and other healthcare data. - International consumer health terminology: This is a Data Object SBB that represents an international dictionary of health terminology. It provides standardized terms and definitions for describing various aspects of consumer health. - International Clinical Terminology for Primary Care: This is a Data Object SBB that represents a set of definitions of terms used to describe the process of primary care. It provides standardized terminology for primary care practitioners. - Logical Observation Identifiers Names and Codes (LOINC): This is a Data Object SBB that represents a database and universal standard for identifying medical laboratory observations. It provides a standardized way of identifying and exchanging laboratory test results. 2. Health Business Information Base: - Diagnosis: This is a Business Object ABB that represents information related to the diagnosis of a patient. It includes details specified by operators, such as symptoms and medical history. - Laboratory: This is a Business Object ABB that represents information about laboratory exams performed by a patient in a healthcare facility. It includes data related to various laboratory tests. - Hospital Enterprise Resource Planning: This is a Business Object ABB that represents information about the management of hospital resources for the execution of health processes. It includes data related to surgery scheduling, beds availability, consultation scheduling, medication and examination scheduling, and general health facility information. The motivation behind this view is to provide a comprehensive framework for leveraging various controlled vocabularies and information related to health. By using standardized terminologies and vocabularies, healthcare organizations can ensure interoperability, improve data quality, and enhance communication and collaboration in the health domain.
